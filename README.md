# BigBoom - 压测日志关键信息分析工具

基于 PyQt6 的桌面 GUI 应用，用于分析嵌入式设备（物联网/智能家居）压测日志，提取关键打印、计算关键时间间隔、检测异常行为，并生成 PDF/Excel 报告。

## 功能

### 1. 关键打印检测
- 根据可配置的正则规则匹配日志行
- 自动识别循环边界，统计完整/不完整循环数及成功率
- 将完整循环提取为独立文件至 `files/<测试项>/` 目录

### 2. 关键时间计算
- 在完整循环内按起止正则配对计算耗时（支持顺序配对/最近匹配两种策略）
- 与参考时间对比，标记超时项
- 支持动态参考时间（按捕获组值切换不同阈值）

### 3. 异常行为检测
- 按测试类别扫描日志中的异常关键字
- 编译为交替正则进行快速预过滤
- 报告每种异常的出现次数、时间戳和行号
- 支持"仅计数"模式合并高频匹配项

### 4. 开关机专项分析
- 硬关机 vs 软关机分类统计（根据 reset reason）
- 电池电量变化折线图
- 电压变化散点图（含 4.2V 参考线）
- 升级版本提取（主控固件/WiFi 固件）

### 5. 多设备批量分析
- 打开多设备文件夹，依次对每个设备执行全部三类分析
- 汇总结果到统一树形视图，每个设备单独导出 Excel 表，PDF 汇总所有设备

### 6. 实时监听模式
- 持续轮询日志文件（可配间隔 30 分钟 ~ 24 小时）
- 文件大小增量追踪，高效重读大日志

### 7. 日志合并
- 双文件合并（堆归并）或按时间戳合并文件夹内所有日志
- 支持 CPU/WiFi 跨类型日志合并

### 8. 导出
- **PDF**: 格式化报告，含水印、页眉页脚、中文字体（微软雅黑）
- **Excel**: 分析结果分 sheet 导出，多设备每设备一个 sheet

## 安装与运行

### 从源码运行

```bash
pip install PyQt6 reportlab matplotlib openpyxl
python main.py
```

### 打包为 EXE

```bash
pip install pyinstaller
pyinstaller --clean BigBoom.spec
```

输出: `dist/BigBoom.exe`（无控制台窗口）

## 配置

`config/` 目录下的 JSON 文件（首次运行自动生成默认值）:

| 文件 | 内容 |
|------|------|
| `config.json` | 用户配置，当前选中的测试项 |
| `rules.json` | 关键字匹配规则：[正则, 含义, 启用, 阶段] |
| `time_rules.json` | 时间计算规则：[测试项, 起正则, 止正则, 参考时间(秒), 取最小值] |
| `anomaly_rules.json` | 异常检测规则：[测试项, 关键字, 含义, 启用, 仅计数] |
| `device_info.json` | 设备信息提取规则：[字段, 正则, 提取值] |

规则可通过 GUI 菜单栏 → 关键字/时间/异常 → 打开对应编辑器进行修改。

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+O` | 打开单个日志文件 |
| `Ctrl+D` | 打开文件夹（合并日志） |
| `Ctrl+Shift+D` | 打开多设备文件夹 |
| `Ctrl+Shift+S` | 保存计算结果 |

## 目录结构

```
BigBoom/
├── main.py                    # 程序入口
├── log_key_info_analize.py    # 主逻辑 (GUI + 分析引擎)
├── BigBoom.spec               # PyInstaller 打包配置
├── requirements.txt           # 依赖
├── config/                    # 运行时配置 (JSON)
├── test/                      # 测试日志文件
├── test_multi_device/         # 多设备测试数据
└── files/                     # 分析输出 (循环提取)
```
